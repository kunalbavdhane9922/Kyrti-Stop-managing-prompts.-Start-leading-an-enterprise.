# Conventions

Date: 2026-06-06

As there is no source code yet, code-level conventions (naming, style, error handling) are not established in practice.

## Documentation Conventions
- Numbered prefixes for documentation directories (e.g., `01-product`, `02-governance`).
- Markdown files with Title Case or Kebab Case naming (e.g., `SystemArchitecture.md`, `DatabaseDesign.md`).

## Principles
- **Human Authority**: AI assists, humans govern.
- **Explainability**: No black-box governance.
- **Auditability**: Traceable workflows and decisions.
- **Sovereignty**: Ownership of data.
